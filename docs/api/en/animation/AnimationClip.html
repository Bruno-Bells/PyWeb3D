[name]

An [name] is a reusable set of keyframe tracks which represent an animation.

For an overview of the different elements of the three.js animation system see the "Animation System" article in the "Next Steps" section of the manual.

Constructor

[name]( [param:str name], [param:int duration], [param:list tracks] )

[page:str name] - a name for this clip.
[page:int duration] - the duration of this clip (in seconds). If a negative value is passed, the duration will be calculated from the passed `tracks` list.
[page:list tracks] - a list of [page:KeyframeTrack KeyframeTracks].
[page:int blendMode] - defines how the animation is blended/combined when two or more animations are simultaneously played.

Note: Instead of instantiating an AnimationClip directly with the constructor, you can use one of its static methods to create AnimationClips: from JSON ([page:.parse parse]), from morph target sequences ([page:.CreateFromMorphTargetSequence CreateFromMorphTargetSequence], [page:.CreateClipsFromMorphTargetSequences CreateClipsFromMorphTargetSequences]) or from animation hierarchies ([page:.parseAnimation parseAnimation]) - if your model doesn't already hold AnimationClips in its geometry's animations list.

Properties

[property:int blendMode]

Defines how the animation is blended/combined when two or more animations are simultaneously played. Valid values are *NormalAnimationBlendMode* (default) and *AdditiveAnimationBlendMode*.

[property:int duration]

The duration of this clip (in seconds). This can be calculated from the [page:.tracks tracks] list via [page:.resetDuration resetDuration].

[property:str name]

A name for this clip. A certain clip can be searched via [page:.findByName findByName].

[property:list tracks]

A list containing a [page:KeyframeTrack] for each property that are animated by this clip.

[property:str uuid]

The [link:http://en.wikipedia.org/wiki/Universally_unique_identifier UUID] of this clip instance. It gets automatically assigned and shouldn't be edited.

Methods

[method:AnimationClip clone]()

Returns a copy of this clip.

[method:self optimize]()

Optimizes each track by removing equivalent sequential keys (which are common in morph target sequences).

[method:self resetDuration]()

Sets the [page:.duration duration] of the clip to the duration of its longest [page:KeyframeTrack].

[method:dict toJSON]()

Returns a JSON object representing the serialized animation clip.

[method:self trim]()

Trims all tracks to the clip's duration.

[method:bool validate]()

Performs minimal validation on each track in the clip. Returns `True` if all tracks are valid.

Static Methods

[method:list CreateClipsFromMorphTargetSequences]( [param:str name], [param:list morphTargetSequence], [param:int fps], [param:bool noLoop] )

Returns a list of new AnimationClips created from the morph target sequences of a geometry, trying to sort morph target names into animation-group-based patterns like "Walk_001, Walk_002, Run_001, Run_002 ...".

[method:AnimationClip CreateFromMorphTargetSequence]( [param:str name], [param:list morphTargetSequence], [param:int fps], [param:bool noLoop] )

Returns a new AnimationClip from the passed morph targets list of a geometry, taking a name and the number of frames per second.

Note: The fps parameter is required, but the animation speed can be overridden in an `AnimationAction` via [page:AnimationAction.setDuration animationAction.setDuration].

[method:AnimationClip findByName]( [param:dict objectOrClipArray], [param:str name] )

Searches for an AnimationClip by name, taking as its first parameter either a list of AnimationClips, or a mesh or geometry that contains a list named "animations".

[method:AnimationClip parse]( [param:dict json] )

Parses a JSON representation of a clip and returns an AnimationClip.

[method:AnimationClip parseAnimation]( [param:dict animation], [param:list bones] )

Parses the animation.hierarchy format and returns an AnimationClip.

[method:dict toJSON]( [param:AnimationClip clip] )

Takes an AnimationClip and returns a JSON.
