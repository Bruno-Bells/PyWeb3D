No [name]

You can use PyWeb3D without having to install anything. There are three steps you need to follow.

Step 1

Adding the necessary libraries and packages into the `<head></head>` tag of your HTML file.

<script src="https://cdn.jsdelivr.net/npm/brython@3.10.7/brython.min.js"></script> <script src="https://cdn.jsdelivr.net/npm/brython@3.10.7/brython_stdlib.js"></script> <script src="https://unpkg.com/three@0.145.0/build/three.js"></script> <script src="https://www.pyweb3d.org/pyweb3d/v1.0.0/pyweb3d.brython.js"></script>

Those are the four necessary scripts you'll need in order to successfully use pyweb3D. The first two scripts will import Brython, the third script will import three.js and the last script will import PyWeb3D

Step 2

The second step is to add `onload="brython()"` into the opening `<body>` tag of your HTML file as an attribute.
So the opening `<body>` tag will look like this one below: <body onload="brython()"> ... </body> This second step tells the browser to call the brython function when the page load.

Note: During development phase you can pass an argument to the `brython()` function, like this: `onload="brython(1)"` or `onload="brython(2)"`. `1` is to have the error messages displayed to the web browser console. `2` is to also get the Javascript code displayed along with the error.

Step 3

The last step is to add a `<script type="text/python"> </script>` tag inside the `<body> </body>` tags where your python code will live. There are two ways to do this:
First method: ( without `src` attribute) <body onload="brython()"> <script type="text/python"> # Your Python Code will live here </script> </body> or second method: (with `src` attribute) <body onload="brython()"> <script type="text/python" src="your-python-file.py"></script> </body>

Importing Packages

After the above steps, you'll be ready to import the packages into your code, here's how to import pyweb3D and other necessary modules from browser import document, window from pyweb3d.pyweb3d import * Note: browser is a brython package (an instance of your browser). Please refer to [link:https://brython.info/static_doc/en/intro.html brython's documentation] to learn more about it's packages

Putting it all together

<!DOCTYPE html> <html> <head> <meta charset="utf-8"> <title>My first pyWeb3D app</title> <script src="https://cdn.jsdelivr.net/npm/brython@3.10.7/brython.min.js"></script> <script src="https://cdn.jsdelivr.net/npm/brython@3.10.7/brython_stdlib.js"></script> <script src="https://unpkg.com/three@0.145.0/build/three.js"></script> <script src="https://www.pyweb3d.org/pyweb3d/v1.0.0/pyweb3d.brython.js"></script> <style> body { margin: 0; } </style> </head> <body onload="brython()"> <script type="text/python"> from browser import document, window from pyweb3d.pyweb3d import * ... </script> </body> </html>

Run a local server

The HTML files can be opened in the browser, but it is recommended to start a web server in the application directory.

Plugins for popular code editors Some code editors have plugins which will spawn a simple server on demand.

  • [link:https://marketplace.visualstudio.com/items?itemName=yandeu.five-server Five Server] for Visual Studio Code.
  • [link:https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer Live Server] for Visual Studio Code.
  • [link:https://atom.io/packages/atom-live-server Live Server] for Atom.

Python server

if you're not using any of the above code editors, you can spin up the python server by running this in a command line(from your working directory)

//Python 2.x python -m SimpleHTTPServer //Python 3.x python -m http.server

Local [name]

If you prefer to install pyWeb3D locally, you will need to install or download brython, three.js and pyweb3D. We will go with the download method to keep things simple.

download links:

  • brython.js
  • brython_stdlib.js
  • three.js
  • pyweb3D

Download and add the files to your working directory, then load the files into the `<head> </head>` tag of your HTML file using the `<script src=""></script>` tags. <script type="text/javascript" src="brython.min.js"></script> <script type="text/javascript" src="brython_stdlib.js"></script> <script type="text/javascript" src="three.js"></script> <script type="text/javascript" src="pyweb3d.brython.js"></script>
