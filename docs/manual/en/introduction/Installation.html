You can use PyWeb3D without having to install anything. There are three steps you need to follow.

Step 1

The first step is to add the necessary libraries and packages into the `<head></head>` tag of your HTML file.

<script src="https://cdn.jsdelivr.net/npm/brython@3.10.7/brython.min.js"></script> <script src="https://cdn.jsdelivr.net/npm/brython@3.10.7/brython_stdlib.js"></script> <script defer src="https://unpkg.com/three@0.145.0/build/three.js"></script> <script defer src="https://www.pyweb3d.org/package/v1.0.0/pyweb3d.brython.js"></script>

Those are the four necessary scripts you'll need in order to successfully use pyweb3D. The first two scripts will import Brython, the third script will import three.js and the last script will import PyWeb3D

Step 2

The second step is to add `onload="brython()"` into the opening `<body>` tag of your HTML file as an attribute.
So the opening `<body>` tag will look like this one below: <body onload="brython()"> ... </body> This second step tells the browser to call the brython function when the page load. During development phase you can pass an argument to the `brython()` function, like this: `onload="brython(1)"` or `onload="brython(2)"`. `1` is to have the error messages displayed to the web browser console. `2` is to also get the Javascript code displayed along with the error.

Step 3

The last step is to add a `<script></script>` tag inside the `<body></body>` where your python code will live
There are two ways to do this:
First method: ( without `src` attribute) <body onload="brython()"> <script type="text/python"> # Your Python Code will live here </script> </body> or second method: (with `src` attribute) <body onload="brython()"> <script type="text/python" src="your-python-file.py"></script> </body>

Note: If you're using the second method, make sure that you python file is in the same folder or domain as your main HTML file
